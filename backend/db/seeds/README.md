# Database Seeds

This directory contains seed data for development and testing of the JusMonitor CRM system.

## Overview

The seed system creates realistic demo data including:

- **Tenant & Users**: Demo law firm with admin, lawyer, and assistant users
- **CRM Data**: 20 leads across different funnel stages and 10 clients with interactions
- **Legal Cases**: 15 judicial processes with realistic CNJ numbers
- **Case Movements**: 100+ process movements with AI analysis
- **AI Configuration**: Multiple AI providers (OpenAI, Anthropic, Groq) with priorities

## Usage

### Seed All Data

```bash
cd backend
python -m cli.seed --all
```

### Seed Specific Components

```bash
# Only tenant and users
python -m cli.seed --tenant

# CRM data (requires tenant)
python -m cli.seed --crm

# Legal cases (requires tenant and CRM)
python -m cli.seed --cases

# AI configuration (requires tenant)
python -m cli.seed --ai

# Combine multiple
python -m cli.seed --tenant --crm --cases
```

### Reset Database

⚠️ **WARNING**: This will delete ALL data!

```bash
python -m cli.seed --all --reset
```

## Seed Files

### `tenant.py`
Creates the demo tenant "Demo Law Firm" and three users with different roles.

**Fixed Tenant ID**: `00000000-0000-0000-0000-000000000001`

### `crm.py`
Creates leads and clients with:
- 20 leads distributed across funnel stages (New → Negotiation)
- 10 clients with notes and timeline events
- AI-generated summaries and recommended actions
- Realistic Brazilian names, phones, and emails (via Faker)

### `legal_cases.py`
Creates legal cases and movements with:
- 15 judicial processes with realistic CNJ numbers
- Various case types (Labor, Civil, Family, etc.)
- 5-15 movements per case
- AI importance classification
- Content hashing for deduplication

**CNJ Number Format**: `NNNNNNN-DD.AAAA.J.TR.OOOO`

### `ai_config.py`
Configures AI providers with:
- Multiple models per provider
- Priority-based routing (OpenAI → Anthropic → Groq)
- Encrypted API keys
- Rate limits and temperature settings

## Notes

### Embeddings
The seed system creates movement records but **does not generate actual embeddings**. In production, embeddings would be generated by:

1. EmbeddingService with OpenAI `text-embedding-3-small`
2. Taskiq worker for async processing
3. pgvector for storage and similarity search

To generate embeddings after seeding, run the embedding worker.

### API Keys
Demo seeds use placeholder API keys. For functional AI features, provide real API keys via environment variables.

### Multi-Tenancy
All seeded data is associated with the demo tenant ID. The system enforces tenant isolation at the database and application levels.

## Troubleshooting

### Import Errors
Ensure you're running from the backend directory:
```bash
cd backend
python -m cli.seed --all
```

### Database Connection
Check your `.env` file has correct database credentials:
```
DATABASE_URL=postgresql+asyncpg://user:pass@localhost/jusmonitor
```

### Missing Dependencies
Install all dependencies:
```bash
poetry install
```

### Permission Errors
Ensure database user has CREATE/DROP permissions for seeding.

## Development

To add new seed data:

1. Create a new seed file in `db/seeds/`
2. Follow the pattern of existing seeds
3. Add to `cli/seed.py` command options
4. Update this README

Example seed function:

```python
async def seed_my_data(
    session: AsyncSession,
    tenant_id: UUID
) -> list[MyModel]:
    """Create demo data."""
    items = []
    
    for i in range(10):
        item = MyModel(
            tenant_id=tenant_id,
            name=f"Item {i}",
        )
        session.add(item)
        items.append(item)
    
    await session.flush()
    return items
```
